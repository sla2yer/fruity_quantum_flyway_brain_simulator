Implemented the Milestone 15 suite planner end to end.

The main addition is a canonical planner API in [experiment_suite_planning.py](/home/jack/Documents/github/personal/fly_neural_simulation/flywire_wave_repo/src/flywire_wave/experiment_suite_planning.py#L259) and [experiment_suite_planning.py](/home/jack/Documents/github/personal/fly_neural_simulation/flywire_wave_repo/src/flywire_wave/experiment_suite_planning.py#L301). `normalize_experiment_suite_config(...)` now normalizes local suite defaults, and `resolve_experiment_suite_plan(...)` resolves either a suite manifest or an experiment manifest with an embedded `suite` block into one deterministic orchestration surface. The returned plan records active dimensions, sweep axes, cross-product vs linked expansion, seed reuse policy, ablation declarations, comparison pairings, stage targets, output roots, and stable suite/work-item/artifact ordering while reusing the existing simulation/readout-analysis planning layers rather than bypassing them. Clear failures were added for unknown dimensions, linked-axis mismatches, unsupported ablation variants, contradictory perturbation seed rules, missing subset prerequisites, and unsupported wave-dependent requests.
